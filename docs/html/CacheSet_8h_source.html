24 /*
25  A Set-associative cache, is a many-to-few mapping between addresses and
26  storage locations. On each lookup, a subset of address bits is used to
27  generate an index, just as in the direct-mapped case. However, this index
28  now corresponds to a set of entries, usually two to eight, that are searched
29  in parallel for a matching tag. In practice, this approach is much more
30  efficient from a hardware implementation perspective, since it requires
31  fewer address comparators than a fully associative cache, but due to its
32  flexible mapping policy behaves similarly to a fully associative cache.
33 
34  Set-associative caches permit the flexible placement of data among all the
35  entries of a set.
36 
37  - index bits - select a particular set,
38  - tag bits - select an entry within the set,
39  - block offset bits - select the word within the selected entry.
40 */

50 // In a 4-way set-associative data cache, the following data is used:
51 //
52 // Total Number of Blocks (b) = 16; (see requirements)
53 // BlockSize (in bytes) = 32; (see requirements)
54 // Number of Blocks per Set (N) = 4; (see requirements) 4-way set associative
55 //
56 // Calculate other items as follows:
57 // ---------------------------------
58 // Number of Sets (s) = b / N;
59 // s = 16 / 4;
60 // s = 4;
61 //
62 // Offset Bits (o) - allows Selecting the word within each block
63 // o = log2 (BlockSize)
64 // o = log2(32)
65 // o = 5;
66 //
67 // Index Bits (i) = allows Selecting the set of blocks
68 // i = log2 (Number of Sets)
69 // i = log2 (4)
70 // i = 2
71 //
72 // Tag Bits (t) = ID blocks within a set
73 // t = sizeof(address) - o - i;
74 // (given a 32-bit address)
75 // t = 32 - 5 - 2;
76 // t = 25;
77 //
